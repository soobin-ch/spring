API Key /Session based Authentication

JWT: 클레임이라고 불리는 정보를 JSON 형태로 안전하게 전송하기 위한 토큰 기반의 
-인증과 정보 교환에 사용, 서명이 되어 있어서 신뢰성 확보가 가능


1. Header  : 토큰의 타입과 사용된 알고리즘 정보를 담고 있음, base64Url 로 인코딩
2.playload : 클레임 정보, 대상, 발행자, 만료 시간 등 다양한 정보가 포함됨, base64Url 로 인코딩
3. Signature : Header 와 playload , secretKey 를 사용한 서명

인증, 정보교환
